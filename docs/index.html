Como ler este relatório

Este painel resume tempo, volume e estabilidade das chamadas feitas pelo JMeter. A ideia é mostrar rapidamente onde a API respondeu bem, onde houve lentidão e em quais pontos Spring e Python se comportaram de forma diferente.

Total de amostras: quantidade de requisições medidas naquele cenário ou endpoint.

Erro: percentual de requisições que falharam. Quanto mais perto de 0%, melhor.

Throughput: quantas requisições por segundo o cenário conseguiu sustentar.

Avg: tempo médio de resposta. Dá uma noção geral, mas pode esconder picos.

p95: tempo abaixo do qual ficaram 95% das respostas. É um bom indicador de experiência real.

p99: mostra caudas de lentidão mais raras, úteis para identificar picos.

Max: maior tempo observado. Serve como alerta, mas isoladamente pode ser ruído.

Latency: tempo até começar a receber a resposta.

Connect: custo médio para abrir a conexão com o banco ou serviço HTTP.

Payload: volume médio de dados trafegados na resposta. Payload maior pode aumentar o tempo.

Delta: diferença entre Python e Spring no mesmo endpoint. Valor positivo indica Python mais lento; negativo indica Python mais rápido.
